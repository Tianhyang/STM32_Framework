## 2026-03-28

### Framework Baseline Completed

当前工程已经完成从 `0` 到 `1` 的可运行框架搭建，主要包括：

- `STM32F103RCT6` 启动、链接、CMSIS、HAL 基线
- `FreeRTOS` 基础接入
- 板级初始化、时钟配置、IRQ 路由
- `LED`、`KEY`、`UART` 基础驱动
- 统一日志接口
- 统一应用事件队列
- 基础软件定时器服务
- 按键 EXTI 唤醒骨架

### Verified Hardware Paths

当前已验证的硬件路径：

- `PC2`、`PC3` 双 LED 正常工作
- `USART1` 串口日志正常输出
- `PA0`、`PC13` 按键输入正常
- `ST-Link` 可通过 `make flash` 烧录

### Important Architecture Outcomes

当前框架已经形成这些核心结构：

- `app`
  - 放应用逻辑、任务、事件处理、服务编排
- `drivers/interface`
  - 放上层依赖的抽象驱动接口
- `drivers/bsp`
  - 放基于当前板子的 STM32 驱动实现
- `platform/boards`
  - 放板级配置、时钟、IRQ 路由
- `rtos`
  - 放 FreeRTOS 配置和项目适配层

### Latest Change

本轮最后完成的改动是：

- 日志输出从阻塞式 UART 发送，切换为 `UART + DMA` 异步发送
- 新增 `DMA1_Channel4` 中断路由
- 保持 `log_info()` / `log_printf()` 上层接口不变

### Recent Fix

本轮补充修正了一处按键消抖逻辑问题：

- 原先按键确认依赖“扫描周期 + 连续计数”
- 当 `APP_KEY_SCAN_PERIOD_MS` 被调大时，短按可能无法通过消抖确认
- 现已改为“检测到状态变化后，固定等待短时间再复查”的时间型消抖
- 修正后，按键识别不再直接受扫描周期放大影响

本轮同时整理了一处 UART 配置一致性问题：

- `s_uart_hw[]` 中的 UART 实例和 TX/RX 引脚信息，改为统一引用 `board_config.h` 中的板级宏
- 避免了在驱动内部重复写死 `USART1`、`GPIOA`、`GPIO_PIN_9/10`
- 让 UART 硬件描述与板级配置保持单一来源
